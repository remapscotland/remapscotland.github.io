    This Document

    Just a simple guide to help simplify things so following the progress of the work is a clear and straightforward job.

    Q. What is the repository anyway?

    A. Essentially it is just a folder containing a 'package' in our case the package is a web application which will contain the website and database archives of REMAP Scotland and its cases. The main branch is often referred to as 'master'... Shall we move on?

    The 'web' Repository

    web This is where the work you see published on http://remapscotland.org.uk goes. Nothing should go on the site without being pushed to 'master' on this repository

    The 'wip' Repository

    wip The wip repository is for 'work in progress'. The site is too young to start using it yet but expect most work to move there once we have a structure which complies with accessibility standards and holds enough content to qualify as a suitable front end.

    The 'remapGit' Repository

    remapGit This repository is for miscellaneous useful scripts. I could not think of a good name at that time, sorry.
